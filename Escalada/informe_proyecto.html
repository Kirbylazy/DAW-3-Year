¿Qué es Laravel? El framework MVC

El patrón de arquitectura que organiza toda la aplicación

¿Qué es un framework?

Un framework es un conjunto de herramientas, convenciones y código ya escrito que te ahorra tener que construir todo desde cero. Laravel es el framework PHP más popular del mundo: te proporciona un sistema de rutas, acceso a base de datos, autenticación, generación de HTML y muchísimo más, para que tú te concentres solo en la lógica de tu aplicación.

El patrón MVC (Modelo - Vista - Controlador)

Laravel sigue el patrón MVC, que divide la aplicación en tres capas con responsabilidades bien separadas:

Capa¿Qué hace?En nuestro proyecto
Modelo (M)Gestiona los datos y la base de datosapp/Models/Competicion.php, User.php, etc.
Vista (V)Genera el HTML que ve el usuarioresources/views/ (archivos .blade.php)
Controlador (C)Recibe la petición, usa el modelo y devuelve la vistaapp/Http/Controllers/

¿Por qué PHP y no otro lenguaje?

PHP fue diseñado específicamente para la web. Laravel añade una capa de elegancia sobre PHP haciendo que el código sea más limpio, organizado y fácil de mantener. La versión que usamos, PHP 8.2, incluye características modernas como tipado estricto, enums, fibers y mucho más.

La base de datos: migraciones y Eloquent ORM

Cómo Laravel gestiona la estructura y los datos de MySQL

¿Qué son las migraciones?

Las migraciones son archivos PHP que describen la estructura de las tablas de la base de datos. En lugar de escribir SQL directamente, describes las tablas en PHP y Laravel las crea automáticamente. Esto permite que cualquier desarrollador reconstruya la BD con un solo comando: php artisan migrate.

✅ Ventaja clave: Si añades una columna nueva a una tabla, creas una migración nueva y todos los demás miembros del equipo la reciben al hacer git pull + php artisan migrate. No hay que compartir scripts SQL manualmente.

Ejemplo: Migración de competiciones

// database/migrations/2026_02_16_150333_create_competicions_table.php
Schema::create('competicions', function (Blueprint $table) {
    $table->id();                         // Columna id autoincremental

    $table->foreignId('copa_id')          // Clave foránea a tabla "copas"
          ->nullable()                    // Puede ser null (sin copa)
          ->constrained('copas')          // Referencia a copas.id
          ->nullOnDelete();               // Si se borra la copa → null (no cascade)

    $table->foreignId('arbitro_id')       // El árbitro asignado
          ->nullable()
          ->constrained('users')          // Referencia a users.id
          ->nullOnDelete();

    $table->foreignId('ubicacion_id')     // Rocódromo donde se celebra
          ->constrained('ubicacions')
          ->cascadeOnDelete();            // Si se borra el rocódromo → borra esto tb

    $table->string('name');               // Nombre de la competición
    $table->string('provincia');          // Provincia
    $table->dateTime('fecha_realizacion');// Fecha y hora de inicio
    $table->dateTime('fecha_fin')->nullable(); // Fecha fin (puede ser null)
    $table->string('tipo');               // bloque / dificultad / velocidad
    $table->boolean('campeonato')->default(false); // ¿Es campeonato?
    $table->json('categorias')->nullable(); // Array JSON de categorías
    $table->timestamps();                 // created_at y updated_at automáticos
});

¿Qué es Eloquent ORM?

ORM significa Object-Relational Mapping. Es un sistema que te permite trabajar con la base de datos usando objetos PHP en lugar de SQL puro. Eloquent es el ORM de Laravel.

// Sin Eloquent (SQL puro — tedioso):
$result = DB::select("SELECT * FROM competicions WHERE id = 5");

// Con Eloquent (mucho más limpio):
$competicion = Competicion::find(5);
echo $competicion->name;          // Accedes a columnas como propiedades
echo $competicion->copa->name;    // ¡Accedes a relaciones automáticamente!
echo $competicion->inscripciones->count(); // Cuenta relaciones
💡 Concepto clave: En Eloquent, cada tabla de la base de datos corresponde a una clase PHP (Modelo). Cada fila de la tabla corresponde a un objeto de esa clase. Modificar el objeto y llamar a save() actualiza la fila en la BD automáticamente.

Los Modelos: el corazón de los datos

Cómo representamos y relacionamos las entidades del dominio

Modelo User — El usuario del sistema

Es el modelo más complejo. Un User puede tener múltiples roles con comportamientos distintos:

class User extends Authenticatable
{
    protected $fillable = [
        'name', 'email', 'password', 'dni',
        'fecha_nacimiento', 'provincia', 'talla', 'genero',
        'rol',   // 'competidor' | 'entrenador' | 'arbitro' | 'admin'
    ];

    // $hidden oculta estos campos cuando convertimos a JSON/array
    protected $hidden = ['password', 'remember_token'];

    // Métodos de comprobación de rol
    public function isAdmin():      bool { return $this->nivelRol() >= 4; }
    public function isArbitro():    bool { return $this->nivelRol() >= 3; }
    public function isEntrenador(): bool { return $this->nivelRol() >= 2; }
    public function isCompetidor(): bool { return $this->nivelRol() >= 1; }

    // Nivel numérico del rol (para jerarquía)
    public function nivelRol(): int {
        return match($this->rol) {
            'admin'      => 4,
            'arbitro'    => 3,
            'entrenador' => 2,
            default      => 1,  // competidor
        };
    }
}

Relaciones del modelo User

Las relaciones son la parte más potente de Eloquent. Definen cómo se conectan los modelos entre sí:

MétodoTipoDescripción

Los tipos de relaciones en Eloquent

HasMany (uno a muchos)

// Una Copa tiene MUCHAS Competiciones
class Copa extends Model {
    public function competiciones(): HasMany {
        return $this->hasMany(Competicion::class, 'copa_id');
    }
}
// Uso:
$copa = Copa::find(1);
$competiciones = $copa->competiciones; // Devuelve una Collection de Competicion

BelongsTo (muchos a uno, inversa)

// Una Competicion PERTENECE a una Copa
class Competicion extends Model {
    public function copa(): BelongsTo {
        return $this->belongsTo(Copa::class, 'copa_id');
    }
}
// Uso:
$competicion = Competicion::find(1);
echo $competicion->copa->name; // "Copa de Bloque 2026"
echo $competicion->copa?->name; // Con "?" por si copa_id es null

BelongsToMany (muchos a muchos)

// Un User puede estar en MUCHAS Competiciones,
// y una Competicion puede tener MUCHOS Users
// Se necesita tabla intermedia: competicions_users
class User extends Model {
    public function competiciones(): BelongsToMany {
        return $this->belongsToMany(
            Competicion::class,
            'competicions_users',   // nombre de la tabla pivote
            'user_id',              // FK de este modelo en la pivote
            'competicion_id'        // FK del otro modelo en la pivote
        )->withPivot('tipoDato', 'dato') // columnas extra de la pivote
         ->withTimestamps();
    }
}

El modelo Inscripcion — lógica de negocio avanzada

El modelo Inscripcion tiene lógica de negocio propia, algo muy común en Laravel:

public function recalcularEstado(): void
{
    // Si algún documento es inválido → rechazada
    if ($this->licencia_estado === 'no_valida' || $this->pago_estado === 'no_valida') {
        $this->estado = 'rechazada';
    }
    // Si ambos documentos son válidos → aprobada
    elseif (
        in_array($this->licencia_estado, ['valida', 'valida_dia']) &&
        in_array($this->pago_estado,     ['valida', 'valida_dia'])
    ) {
        $this->estado = 'aprobada';
    }
    // Si no → pendiente
    else {
        $this->estado = 'pendiente';
    }
    $this->save(); // Guarda en la BD
}

// Calcula la categoría automáticamente según la edad y género del competidor
public static function calcularCategoria(User $user): string
{
    $edad = Carbon::parse($user->fecha_nacimiento)->age;
    $cat  = match(true) {
        $edad <= 8  => 'U9',
        $edad <= 10 => 'U11',
        $edad <= 12 => 'U13',
        $edad <= 14 => 'U15',
        $edad <= 16 => 'U17',
        $edad <= 18 => 'U19',
        $edad <= 34 => 'Absoluta',
        default     => 'Veterana',
    };
    $genero = match($user->genero) {
        'M' => 'Masculino',
        'F' => 'Femenino',
        default => 'Otro',
    };
    return "$genero $cat"; // Ej: "Femenino U15"
}

$fillable y la protección de asignación masiva

⚠️ Seguridad: En Laravel, $fillable define qué campos pueden asignarse masivamente. Si un usuario malicioso envía rol=admin en un formulario, Laravel lo ignorará si rol no está en $fillable. Esto previene ataques de mass assignment.

Las Rutas: cómo llegan las peticiones

El archivo web.php — el mapa de toda la aplicación

¿Qué es una ruta?

Una ruta asocia una URL y un método HTTP (GET, POST, PATCH, DELETE) con una acción específica del controlador. Todo esto se define en routes/web.php.

// GET /admin/copas → ejecuta AdminController::copas() → devuelve vista
Route::get('/admin/copas', [AdminController::class, 'copas'])->name('admin.copas');

// POST /admin/copas → ejecuta CopaController::store() → guarda nueva copa
Route::post('/admin/copas', [CopaController::class, 'store'])->name('admin.copas.store');

// PATCH /admin/copas/{copa} → ejecuta CopaController::update() → edita copa
Route::patch('/admin/copas/{copa}', [CopaController::class, 'update'])->name('admin.copas.update');

// DELETE /admin/copas/{copa} → ejecuta CopaController::destroy() → borra copa
Route::delete('/admin/copas/{copa}', [CopaController::class, 'destroy'])->name('admin.copas.destroy');
💡 Los métodos HTTP: Los navegadores solo pueden enviar GET y POST. Para PATCH y DELETE, Laravel usa un campo oculto _method en el formulario. El Blade @method('PATCH') lo genera automáticamente.

Rutas con parámetros: Route Model Binding

Cuando una ruta tiene {copa} o {competicion}, Laravel automáticamente busca el registro correspondiente en la BD y lo inyecta en el controlador. A esto se llama Route Model Binding.

// Ruta:
Route::patch('/admin/copas/{copa}', [CopaController::class, 'update']);

// Controlador — Laravel inyecta automáticamente el objeto Copa:
public function update(Request $request, Copa $copa)
{
    // $copa ya es el objeto Copa con id = el número de la URL
    // Si no existe → Laravel devuelve automáticamente un 404
    $copa->update(['name' => $request->name]);
}

Grupos de rutas con middleware y prefijos

En vez de repetir el middleware y prefijo en cada ruta, se agrupan:

// Todas las rutas admin comparten: autenticación + rol admin + prefijo /admin
Route::middleware(['auth', 'rol:admin'])->prefix('admin')->name('admin.')->group(function () {
    Route::get('/pruebas',    [AdminController::class, 'pruebas'])->name('pruebas');
    Route::get('/copas',      [AdminController::class, 'copas'])->name('copas');
    Route::get('/usuarios',   [AdminController::class, 'usuarios'])->name('usuarios');
    Route::get('/rocodromos', [AdminController::class, 'rocodromos'])->name('rocodromos');
    // ...más rutas
});
// Resultado: ruta 'admin.copas' → URL /admin/copas → requiere auth + rol:admin

Rutas con nombre (named routes)

El ->name('admin.copas') asigna un nombre a la ruta. Esto es fundamental porque en las vistas y controladores nunca escribimos URLs hardcodeadas; en su lugar usamos el nombre:

// En una vista Blade — genera la URL correcta automáticamente:
<a href="{{ route('admin.copas') }}">Ver copas</a>
<form action="{{ route('admin.copas.update', $copa->id) }}" method="POST">

// En un controlador — redirige a una ruta con nombre:
return redirect()->route('admin.pruebas')->with('status', 'Copa creada.');
✅ Ventaja: Si cambias la URL de una ruta (de /admin/copas a /panel/copas), solo tienes que cambiarlo en web.php. Todas las vistas que usen route('admin.copas') seguirán funcionando automáticamente.

Los Controladores: lógica de negocio

Cómo procesa Laravel cada petición

¿Qué hace un controlador?

El controlador es el intermediario entre las rutas y los modelos/vistas. Recibe la petición HTTP, valida los datos, consulta o modifica la BD usando los modelos, y devuelve una respuesta (normalmente una vista o una redirección).

AdminController — Panel de administración

// Método para listar y filtrar competiciones (pruebas):
public function pruebas(Request $request)
{
    $filtro  = $request->get('filtro', 'proximas');
    $copaId  = $request->get('copa_id', '');

    // Construimos la query de forma dinámica según los filtros
    $query = Competicion::with('copa', 'ubicacion', 'arbitro')
                        ->withCount('inscripciones');

    // Aplicamos filtro de fecha
    if ($filtro === 'proximas') {
        $query->where('fecha_realizacion', '>=', now());
    } elseif ($filtro === 'este_año') {
        $query->whereYear('fecha_realizacion', now()->year);
    }

    // Aplicamos filtro de copa
    if ($copaId === 'sin_copa') {
        $query->whereNull('copa_id');
    } elseif ($copaId) {
        $query->where('copa_id', $copaId);
    }

    $competiciones = $query->orderBy('fecha_realizacion')->get();

    // Datos necesarios para los formularios
    $copas     = Copa::orderBy('temporada', 'desc')->get();
    $ubicaciones = Ubicacion::orderBy('name')->get();
    $arbitros  = User::where('rol', 'arbitro')->orWhere('rol', 'admin')->get();

    // Devolvemos la vista con todos los datos necesarios
    return view('admin.pruebas', compact('competiciones', 'copas', 'ubicaciones', 'arbitros', 'filtro', 'copaId'));
}

CompeticionController — CRUD de competiciones

public function store(Request $request)
{
    // 1. VALIDACIÓN: Laravel valida automáticamente y devuelve errores si falla
    $request->validate([
        'name'              => 'required|string|max:150',
        'tipo'              => 'required|in:bloque,dificultad,velocidad',
        'fecha_realizacion' => 'required|date',
        'fecha_fin'         => 'nullable|date|after_or_equal:fecha_realizacion',
        'provincia'         => 'required|string',
        'ubicacion_id'      => 'required|exists:ubicacions,id', // debe existir en BD
        'copa_id'           => 'nullable|exists:copas,id',
        'arbitro_id'        => 'nullable|exists:users,id',
        'categorias'        => 'nullable|array',
        'categorias.*'      => 'in:U9,U11,U13,U15,U17,U19,Absoluta,Veterana,Promoción',
    ]);

    // 2. CREACIÓN: Eloquent crea el registro en la BD
    Competicion::create([
        'name'              => $request->name,
        'tipo'              => $request->tipo,
        'fecha_realizacion' => $request->fecha_realizacion,
        'fecha_fin'         => $request->fecha_fin ?: null,
        'provincia'         => $request->provincia,
        'ubicacion_id'      => $request->ubicacion_id,
        'copa_id'           => $request->copa_id ?: null,
        'arbitro_id'        => $request->arbitro_id ?: null,
        'campeonato'        => false,
        'categorias'        => $request->categorias ?? [],
    ]);

    // 3. REDIRECCIÓN con mensaje flash (se muestra una vez)
    return back()->with('status', "Prueba «{$request->name}» creada correctamente.");
}
💡 Mensajes flash: ->with('status', 'mensaje') almacena el mensaje en la sesión durante UNA sola petición. En la vista, session('status') lo recupera. Es el patrón estándar para mostrar confirmaciones tras formularios.

ArbitroController — Validación de inscripciones

El árbitro puede verificar documentos de cada inscripción. Este es el método más complejo del proyecto:

public function validarLicencia(Request $request, Inscripcion $inscripcion)
{
    // Verificamos que este usuario es el árbitro asignado a esta competición
    $this->checkArbitroPorCompeticion($inscripcion->competicion_id);

    $request->validate([
        'tipo'     => 'required|in:licencia,pago',
        'decision' => 'required|in:valida,valida_dia,no_valida',
        'motivo'   => 'required_if:decision,no_valida|nullable|string|max:500',
    ]);

    if ($request->tipo === 'licencia') {
        $inscripcion->licencia_estado = $request->decision;
        $inscripcion->licencia_motivo = $request->decision === 'no_valida'
            ? $request->motivo : null;

        // Si la licencia es válida, registramos en LicenciaValidacion
        // para que no haya que re-validarla en otras competiciones del mismo año
        if ($request->decision !== 'no_valida') {
            $validaHasta = $request->decision === 'valida'
                ? now()->endOfYear()->toDateString()       // válida hasta 31/12
                : $inscripcion->competicion->fecha_realizacion->toDateString(); // solo ese día

            LicenciaValidacion::updateOrCreate(
                ['user_id' => $inscripcion->user_id, 'competicion_id' => $inscripcion->competicion_id],
                ['validada_por' => auth()->id(), 'tipo' => $request->decision, 'valida_hasta' => $validaHasta]
            );
        }
    } else {
        $inscripcion->pago_estado = $request->decision;
        $inscripcion->pago_motivo = $request->decision === 'no_valida' ? $request->motivo : null;
    }

    // Recalcular el estado global de la inscripción
    $estadoAnterior = $inscripcion->estado;
    $inscripcion->recalcularEstado();

    // Notificar al competidor si el estado global cambia
    if ($inscripcion->estado !== $estadoAnterior) {
        $inscripcion->user->notify(new InscripcionActualizadaNotification($inscripcion, $motivo));
    }

    // Responde en JSON (la llamada viene de Alpine.js / fetch)
    return response()->json(['success' => true, 'message' => 'Verificado.']);
}

Protección CSRF

CSRF (Cross-Site Request Forgery) es un ataque donde una web maliciosa hace peticiones en nombre del usuario. Laravel lo previene con un token único por sesión.

⚠️ Atención: Si olvidas @csrf en un formulario POST, Laravel devolverá un error 419 (Page Expired). Es un error muy común al aprender Laravel.

Hash de contraseñas

Las contraseñas nunca se guardan en texto plano. Laravel usa bcrypt (o Argon2) para hacer un hash unidireccional:

// Al hacer login — compara el hash:
Hash::check($request->password, $user->password) // true o false

💡 Jerarquía: Un árbitro tiene nivel 3, lo que significa que también tiene los permisos de entrenador (nivel 2) y competidor (nivel 1). El middleware verifica nivelRol() >= nivelRequerido, no igualdad estricta.

El middleware CheckRol

Un middleware es código que se ejecuta antes de que la petición llegue al controlador. El middleware CheckRol verifica que el usuario tenga el nivel de rol suficiente:

// app/Http/Middleware/CheckRol.php
class CheckRol
{
    public function handle(Request $request, Closure $next, string ...$roles): Response
    {
        $user = auth()->user();

        // Mapa de roles a niveles numéricos
        $niveles = ['competidor' => 1, 'entrenador' => 2, 'arbitro' => 3, 'admin' => 4];

        // Nivel mínimo requerido (el más bajo de los roles pedidos)
        $nivelRequerido = min(array_map(fn($r) => $niveles[$r] ?? 0, $roles));

        // Si el usuario no tiene nivel suficiente → redirige con error
        if (!$user || $user->nivelRol() < $nivelRequerido) {
            return redirect()->route('dashboard')
                             ->with('error', 'No tienes permiso para acceder aquí.');
        }

        return $next($request); // Continúa con la petición
    }
}

Cómo se registra y usa el middleware:

// bootstrap/app.php — registro del middleware con alias:
$middleware->alias(['rol' => CheckRol::class]);

// routes/web.php — uso en grupos de rutas:
Route::middleware(['auth', 'rol:admin'])->group(function () {
    // Solo accesible por admins (nivel 4)
});

Route::middleware(['auth', 'rol:arbitro'])->group(function () {
    // Accesible por árbitros (nivel 3+) y admins (nivel 4)
});

Registro y cambio de roles

Por defecto, los usuarios se registran con rol competidor. Solo el admin puede cambiar los roles desde el panel:

// AdminController@updateRol
public function updateRol(Request $request, User $user)
{
    $request->validate([
        'rol' => 'required|in:competidor,entrenador,arbitro'

    return back()->with('status', "Rol de {$user->name} actualizado.");
}

public function uploadLicencia(Request $request, Competicion $competicion)
{
    $request->validate([
        'licencia' => 'required|file|mimes:pdf,jpg,jpeg,png|max:5120' // max 5MB
    ]);

    $inscripcion = Inscripcion::where('user_id', auth()->id())

Paso 5: El árbitro verifica un documento

El árbitro accede a /arbitro/competicion/{id}, ve la lista de competidores por categoría, abre el modal de verificación y marca la licencia como válida/no válida. La llamada se hace con fetch() de JavaScript (Alpine.js):

}
